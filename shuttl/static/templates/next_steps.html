Welcome to Shuttl!

Here are the next steps to follow to start using Shuttl.

  • Get your domain name pointing to Shuttl

    There are a couple ways to do this. They are all easy and depends on how you want to do this:

    1. Set up a CNAME to point a subdomain to Shuttl

      CNAMES allow you to make an alias for a domain. This offers less control over the way Shuttl is presented to your clients because it only works for subdomains. For example, shuttl.{{ reseller.url }} is ok but {{ reseller.url }}/shuttl is not.

      Create a new CNAME record with your registrar to point a subdomain of your choice to vendors.shuttl.io.

    2. Set up your server(s) as a proxy

      This gives you more control over how Shuttl is presented to your clients. This allows you to make a {{ reseller.url }}/shuttl URL. However, this does require knowledge of server administration and access to your web server configuration files.

      There are still more ways to do this depending if you're using Nginx or Apache.

      Nginx

      First open the NGINX configuration file you want to modify. You will usually need to open them as root. On Ubuntu Linux, the files are usually in the /etc/nginx/sites-enabled directory. You'll want to modify the one that uses affects your website. After you edit it, add these parameters to your configuration file: 

      
	server {
		...
		location /subdir you want shuttl to be on {
			proxy_pass       https://vendor.shuttl.io;
			proxy_set_header Host            $host;
		}
		...
	}

      Apache

      Apache is a little more difficult to set up as a reverse proxy. The great guys over at Digital Ocean made this great guide. Be sure to follow this guide. The url to forward to is https://vendors.shuttl.io

      Microsoft IIS

      Here is a great guide to how to set up the proxy in IIS: Setting up a reverse proxy in IIS Be sure to follow this guide. The url is again to is https://vendors.shuttl.io

  • Install Houston

    Houston is the command center for Shuttl. Houston allows you to create new organizations, create new websites, and sync your files with Shuttl all from your computer. You could also manage your Shuttl settings. If you don't want to use houston, you can always edit your files on Shuttl or you can use git to to deploy your templates.

    Installing Houston is really simple. Simply install Node and NPM and then run npm install --global shuttl-houston

  • Upload your Branding, Set your pricing, and optionally upload storefront templates

    Because Shuttl is completely white labeled, you need to brand the webpage. This is simple to do. Just choose three colors, upload your logo, set the navigation links, and your pricing and you're done branding your store front.

    Want more control over what you're storefront looks like? Just upload some Shuttl templates, CSS, and JavaScript.
